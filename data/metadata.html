Data Policy

EarthScope-Oceans stations rely on financial support from individual organizations and funding agencies.
As such, specific organizations may have different requirements, and as such, individual organizations may determine their own data release policy. Data Release Policy.

Seismic Data

MERMAID continuously records and automatically reports triggered acoustic pressure time series (seismograms). Data requests can be made from a one-year rolling buffer. Contact Joel D. Simon.

MERMAID data are deposited with the Data Management Center of the EarthScope Consortium.
Our International Federation of Digital Seismograph Networks FDSN code is MH. (Floats P0008, P0010, and P0016 are completely unrestricted. Data from other floats is released following standard NSF policy, see below for access and authentication. Data deposited to date. Data openly available.

Users shall reference the FDSN DOI 10.7914/SN/MH.

Real-Time Metadata

MERMAIDs move! To date the miniSEED format only accommodates the deployment dates. It this therefore imperative to download the GeoCSV files which contain all the details of a MERMAID's evolving position. Browse the complete metadata archive. Work is underway to get GeoCSV embedded into StationXMl.

For the example of MERMAID P0006, queries returning the station information in StationXML, the GeoCSV positions, the instrument response plot, and the instrument response in SACPZ and in RESP formats.

Authentication

Access to data from floats P0008, P0010, and P0016 is always unrestricted.

Restricted data are managed by three pieces of information:

  1. The experiment network password is used to verify that a user requesting access to the experiment should be given access. This password is passed from the PIs to users who should be allowed access. Contact Joel D. Simon.
  2. The web-authentication password allows users access to data via Web Services for experiments that they are permitted to view. It is directly coupled with an email address.
  3. The ssl-encryption password allows users to unscramble breq_fast requests to restricted experiments. This password is also coupled to user's email addresses.
If you have the   experiment network password, you can register for data access  here.
